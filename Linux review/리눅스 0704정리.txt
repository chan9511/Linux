리눅스 -> 명령커멘드 -> gedit b.txt 메모장 = a.txt 열기기능
sudo를 쓰면 전체컴퓨터에서 모두 사용할 수 있게 된다.

이름바꿔서 저장 mv./b.txt ./c.txt    ( mv [옵션][이동 할 파일][이동 될 위치] )
다른이름으로 복사하기 cp./b.txt ./c.txt
삭제기능 rm./b.txt

자주 사용하는 옵션
-b : 이동될 파일이 존재하면 백업파일을 만듦
-i : 이동될 파일이 이름이 이미 존재할 경우, 사용자에게 덮어 쓰기 여부를 묻습니다.
-b : 이동될 파일이 이름이 이미 존재할 경우, 백업파일을 생성합니다.
-f :이동 될 파일이 이름이 이미 존재 할 경우, 강제로 덮어쓰기 합니다.
-n :이동 될 파일이 이름이 이미 존재 할 경우, 덮어쓰기를 하지 않습니다.
-r : 하위 디렉토리 까지 모두 이동합니다.
-v : 이동 진행 상태를 출력합니다.

vi란? 'visual editor' (명령어 기반 편집기) GUI 사용x
vi 사용법
명령키
:q	vi에서 작업한 것이 없을 때 그냥 종료
:q!	작업한 내용을 저장하지 않고 종료
:w파일명	작업한 내용을 저장. 파일명을 지정하면 다른 파일로 저장
:wq, :wq!	작업한 내용을 저장하고 vi를 종료
shift+zz	작업한 내용을 저장하고 vi를 종료
 	느낌표는 강제의 의미

vi의 장점: 포팅 가능성(다양한 운영 체제, 플랫폼에서 사용가능), 적은 자원 소비(가볍다), 강력한 기능(편집수월), 시스템 유지보수, 널리 사용되는 표준(호환)

Symbolic Links ( 바로가기와 같은 원리 ) . 심볼 링크라 불리고, 파일 또는 디렉토리를 가리키는
특수한 파일 유형. 경로를 설정해주어 따로 클릭클릭하여 들어갈 필요없이 한 번에 열 수 있음.
